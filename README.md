# Assignment 2 - Input Form Validation

사용자 이름, 이메일, 메시지를 입력받는 폼을 구현하고 **HTML/CSS/JavaScript**를 활용해 입력 유효성 검사, 오류 메시지 표시, 성공 메시지 출력, 반응형 UI를 구현한 프로젝트

---

## 주요 기

### HTML
- 시맨틱 태그를 사용해 구조적 의미를 명확히 구성  
- `label`과 `for` 속성을 통해 **접근성** 강화  
- 필수 입력 필드에 `required` 속성 적용  
- 직관적인 폼 레이아웃 구성  

### CSS
- **반응형 디자인**: 모바일·데스크톱 화면 대응  
- 사용자 경험 향상을 위한 focus, hover, active 상태 스타일 지정  
- 오류 발생 시 **빨간 테두리** 및 안내 문구 표시  
- 제출 성공 시 Alert **성공 메시지 표시**

### JavaScript
- **실시간 입력 유효성 검사 로직** 구현  
- 검증 조건:
  - 이름: 2자 이상  
  - 이메일: 정규식 기반 유효 이메일 검사  
  - 메시지: 10자 이상  
- **제출 버튼 클릭 시** 모든 필드 검증 후 성공 메시지 출력  
- 성공 제출 시:
  - 폼 초기화  
  - 오류 상태 초기화

## 기능 상세

### 입력 유효성 검사

| 필드 | 조건 | 설명 |
|------|------|------|
| 이름 | 2자 이상 | 비어 있거나 2자 미만일 경우 오류 메시지 표시 |
| 이메일 | 유효한 이메일 형식 | 정규식 검사 실패 시 오류 안내 |
| 메시지 | 10자 이상 | 부족할 경우 오류 메시지 표시 |

---

### 오류 처리 UX

- 제출 전에는 **에러 표시 없음**  
- 제출 버튼 클릭 시 유효성 검사 후 **에러 표시**  
- 입력을 다시 시작하면 해당 필드의 **실시간 검증** 반영

---

### 제출 성공 처리

모든 입력이 통과되면:

- 상단에 **"성공적으로 제출되었습니다."** 메시지 표시  
- 3초 후 메시지 자동 숨김  
- 폼 전체 **초기화**  
- 모든 오류 UI **초기화**

---

## 반응형 디자인

- `flexbox` 기반 반응형 레이아웃  
-  뷰포트 기준 레이아웃 정렬
